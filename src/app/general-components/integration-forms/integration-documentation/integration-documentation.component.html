Creating Incoming Webhook
  • Create a new app for your ORG in slack, if you have already slack in your ORG skip this
  • Click here to create new app for Incoming Webhooks
  • After create new app you will get page in that page Under features section select Incoming webhooks and activate the incoming webhooks
  • Then you will get a Webhook URLs for Your Workspace, then click Add New Webhook to Workspace and then select a channel to post a notification and click allow button to allow the notification
  • Create your webhook, While creating a webhook select the channel you want to post the notifications and click authorize button. If you select the private channel you must first be in that channel.
  • If you select the private channel you must first be in that channel
  • Then copy the Webhook URL (EX: https://hooks.slack.com/services/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)
  • Then enter your webhook URL in the slack integration form
  • Slack API Documentation for creating incoming webhook URL- Click here to read the more details

OAuth and Permissions
  • You will get page in that page Under features section select OAuth & Permissions
  • Under the "Scopes" section, find the Bot Token Scopes section and click on Add an OAuth Scope button and add user:read, user:read.email, chat:write
  • Navigate to OAuth Tokens for Your Workspace section and click install to workspace
  • After installation, you will receive a new Bot User OAuth Token with the required scope. Make sure to copy and securely store the generated token.
  • Then enter your Bot User OAuth Token in the slack integration form

Creating Your App
  • You need to create account in Meta Developers
  • If you have facebook account use this account to login
  • Then click get started to Create a App
  • Select an app business type
  • Fill the form details with required information
  • Then you will get the Add products to your app page in this page select a WhatsApp integrate and continue the steps
  • Now you will get the post request URL and Temporary access token URL page
  • Add mobile numbers in "To" field click Manage Phone Number List to add mobile number and you will get OTP to add the mobile number, You can add upto 5 mobile numbers
  • Now choose the added mobile Number, And click the 'Send Meaasage' button below the url section
  • Now you will get the whatsapp notification (Like 'Hello World etc...')
  • Note : You need to send reply for that message (Like 'Hi') , Then only you get the whatsapp integration notifications from tcube
  • If you want to send notifications permanently you need to create Permanent Access Token. In your app side nav click the "configuration" under the "whatsapp section". (Instructions Below)
Creating Permanent Access Token
  • In top nav bar click App Dashboard -> Business settings
  • In users -> System users
  • In system users page click Add button to open create system user, give username and role(choose admin role) create system user
  • Click on the Add Asserts on the current page(System user page). Choose App -> Select App Name -> Slect Full control option & click save changes button
  • Select the recently created system user from the list and then click the “Generate new token” button -> select the app name
  • Then select the Available Permissions: whatsapp_business_messaging, whatsapp_business_management click Generate token to generate the token and copy and save this token in your local machine(it won't be stored at Facebook.)
  • Now you will get the permanent access token

Create an API token
Create an API token from your Atlassian account:
  • Log in to https://id.atlassian.com/manage-profile/security/api-tokens
  • Click Create API token
  • From the dialog that appears, enter a memorable and concise Label for your token and click Create
  • Click Copy to clipboard, then paste the token on the Basic Auth Token form-field
Get Jira Software base URL
  • Fetch the search bar URL in your Jira Software as given by your admin
  • For Example https://tcube.atlassian.net/jira/your-work
  • Trim that link before the '/jira' and you will get like this link https://tcube.atlassian.net
    • Note : Never put a '/' at end of the base URL
  • Copy and paste that link Jira Base URL Form-field
Choose Projects
  • Choose single/multiple projects as you created in Jira Software
  • Please make sure that spelling of the project should be same in both Jira Software and Tcube application
  • Please cross-check these above two conditions, If you won't get the active sprint issues
Create an Access token
Create an Access token from your Gitlab account:
  • Log in to your GitLab account
  • Click Preferences from right side top profile menu
  • Click Access tokens from left side sidenav menu
  • From the appearing page, enter a token name and expiration, in select scopes check api, read_api, read_user, read_repository click create personal access token
  • Click copy access token that appears in top of this section to clipboard, then paste the token on the Access Token form-field
  • Note: Access token creating person repositories only considered, so please create token with all repositories accessible person account
Get GitLab base URL
  • Copy and paste that link GitLab base URL of your organization Form-field, For Example https://git.servx.global
